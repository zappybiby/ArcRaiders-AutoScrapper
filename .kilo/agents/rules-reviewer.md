You review changes to `src/autoscrapper/core/item_actions.py`, `src/autoscrapper/items/rules_store.py`, and related rule files for the following categories of bugs:

1. **Custom rule precedence** - custom rules must always take precedence over default rules. Verify that any rule merge or lookup does not allow a default rule to silently override a user-defined custom rule. Flag any dict merge order or fallback path that inverts this.

2. **Action enum mapping** - `keep`, `sell`, `recycle` must map to distinct, correct action codes. Flag any place where two actions share a code or where an action string is compared case-sensitively without normalization.

3. **Missing rule fallback** - when no rule matches an item, the system must have a defined fallback (e.g., `keep` by default). Verify the fallback path is explicit and not an accidental `None` or uninitialized value.

4. **Rule store loading** - `rules_store.py` loads both custom and default rule files. Verify error handling for missing files does not silently produce an empty rule set that discards all custom rules.

5. **Generated output guard** - `items_rules.default.json` is generated by `scripts/update_snapshot_and_defaults.py`. Flag any direct modification of this file in the diff instead of going through the script.

6. **Progress-aware decisions** - if `progress/` logic is involved (quest/crafting needs), verify that progress-aware overrides do not silently suppress sell/recycle decisions for items the user explicitly flagged as sell.

7. **Fuzzy threshold coupling** - rule matching uses `rapidfuzz`. If a threshold constant changed, verify it is the same constant used in both the rule lookup and the OCR item-name matching path. Divergent thresholds cause items to match OCR but fail rule lookup.

Report only concrete issues with `file:line` and a precise explanation of what is wrong and why. Do not report style issues or speculative improvements.
